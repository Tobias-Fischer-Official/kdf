the install() or uninstall() methods of each active ProductAction object that is associated with the installation ProductTree.

ProductActionSupport contains methods to allow the logging of events that take place. These methods can be used to log events so that they are seen by the end-user after the installation process has completed (in the post-install summary panel).

In addition to logging functionality, this class provides methods that allow each ProductAction access to the uninstaller. These methods are utilized by the ProductAction to affect the resources that are built into the Uninstaller resource archive. Custom ProductActions should use these methods to store any resources that are required when the uninstallation is run, or when an update installation takes place. In addition, these methods should be used to exclude any unecessary resources from the uninstaller to reduce the overhead of the uninstallation archive.

NOTES :

  • When adding classes and packages to the archive, dependencies are not automatically added. Dependent classes and packages need to be explicitly added.

  • General run-time resources are stored using the putUninstallerResource() methods, and are retrieved at run-time using the WizardServices.getResource() or Wizard.getResource() method. These resources are saved separately from the class archive in a dynamic resource archive that is updated each time the installation, or the associated uninstallation, is run to completion. As a result, these resources can be modified at run-time to store transient information for use when the install/uninstall is run again.

    For example, the Files ProductAction utilizes a general run-time resource to store information on the files that it has installed, and uses that information at uninstallation to determine what needs to be removed from the target system.

  • If the creation of a resource is required prior to archive creation, a temporary resource can be created through the use of the FileUtils method. Temporary files created using the methods in the FileUtils class are automatically removed after the installation has completed.

    The ProductActionSupport class also provides access to the MutableOperationState instance that is used to monitor the progress of the install/uninstall operation. A MutableOperationState is used to monitor an asynchronous operation, and provides a ProductAction with a simple way to display its operation progress, as well as a way to monitor the current state of the operation (e.g., suspended, canceled, etc.). A custom ProductAction should use the MutableOperationState instance to update its progress, as well as to monitor whether or not the process should be continued.
